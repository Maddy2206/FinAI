## Architecture Overview

```
User Browser (Next.js 15 App Router)
  ↕ Clerk Auth (JWT)
  ↕ Convex Client (real-time subscriptions)
        ↕ Convex Functions (queries, mutations, actions)
              ↕ Google Vision API  (receipt OCR)
              ↕ Claude API         (AI structuring, assistant, reports)
              ↕ Resend             (email weekly reports)
              ↕ UploadThing        (file storage)
```

**Key architectural decisions:**
- Convex actions (not Next.js API routes) call external APIs — keeps secrets server-side
- Optimistic UI updates via Convex's built-in mutation optimism
- Convex scheduled functions replace external cron for weekly reports
- Claude API called exclusively from Convex actions (never client-side)

## Database Schema (`convex/schema.ts`)

```typescript
// users — synced from Clerk webhooks
users: {
  clerkId: string,
  email: string,
  name: string,
  currency: string,           // default "INR"
  financialHealthScore: number,
  createdAt: number,
}

// expenses — core transaction records
expenses: {
  userId: Id<"users">,
  amount: number,
  category: ExpenseCategory,  // enum: Food|Travel|Shopping|Rent|Utilities|Entertainment|Healthcare|Subscriptions|Other
  description: string,
  date: number,               // unix timestamp
  isRecurring: boolean,
  receiptId?: Id<"receipts">,
  source: "manual" | "ocr" | "nlp",
  createdAt: number,
}

// budgets — monthly budget limits per category
budgets: {
  userId: Id<"users">,
  category: ExpenseCategory,
  monthlyLimit: number,
  month: string,              // "2026-05" format
  createdAt: number,
}

// receipts — uploaded receipt metadata + extracted data
receipts: {
  userId: Id<"users">,
  fileUrl: string,
  uploadThingKey: string,
  rawOcrText: string,
  extractedData: {
    merchant: string,
    totalAmount: number,
    date: string,
    items: Array<{ name: string, price: number }>,
    category: ExpenseCategory,
  },
  status: "processing" | "done" | "failed",
  createdAt: number,
}

// ai_insights — stored AI analysis results
ai_insights: {
  userId: Id<"users">,
  type: "weekly_summary" | "overspending_alert" | "savings_tip" | "anomaly",
  content: string,            // JSON stringified insight
  month: string,
  createdAt: number,
}

// chat_messages — AI assistant conversation history
chat_messages: {
  userId: Id<"users">,
  role: "user" | "assistant",
  content: string,
  createdAt: number,
}

// weekly_reports — generated report records
weekly_reports: {
  userId: Id<"users">,
  weekStart: string,          // "2026-05-11"
  weekEnd: string,            // "2026-05-17"
  reportContent: string,      // AI-generated markdown
  totalSpent: number,
  topCategory: string,
  emailSentAt?: number,
  createdAt: number,
}
```

### Phase 6 — AI Receipt Scanning
**Goal:** Upload receipt photo → auto-populated expense entry.

Pipeline: `ReceiptUploader → UploadThing → Convex action → Google Vision OCR → Claude structuring → DB storage`

1. **UploadThing setup** (`app/api/uploadthing/route.ts` + `lib/uploadthing.ts`):
   - Configure file router: image uploads up to 4MB
   - On upload complete: call Convex mutation to store fileUrl + trigger OCR action
2. **`convex/receipts.ts`** — `processReceipt` action:
   ```typescript
   // Step 1: Call Google Vision API
   const ocrText = await callGoogleVision(fileUrl)  // lib/vision.ts
   
   // Step 2: Send OCR text to Claude
   const extracted = await callClaude(`
     Extract from this receipt OCR text: ${ocrText}
     Return JSON: { merchant, totalAmount, date, items[], category }
     Categories: Food|Travel|Shopping|Rent|Utilities|Entertainment|Healthcare|Subscriptions|Other
   `)
   
   // Step 3: Store in receipts table + auto-create expense
   await ctx.db.patch(receiptId, { extractedData: extracted, status: "done" })
   await ctx.db.insert("expenses", { ...extracted, source: "ocr", receiptId })
   ```
3. **`lib/vision.ts`** — `callGoogleVision(imageUrl: string): Promise<string>`:
   - Uses Google Vision REST API `annotate` endpoint with `TEXT_DETECTION`
   - Returns concatenated text blocks
4. **Receipts page** (`app/(dashboard)/receipts/page.tsx`):
   - Drag-and-drop uploader with `@uploadthing/react`
   - Processing state with skeleton cards
   - Extracted data preview with edit capability before saving

   ```typescript
   import { cronJobs } from "convex/server"
   const crons = cronJobs()
   crons.weekly("weekly-reports", { dayOfWeek: "monday", hourUTC: 8, minuteUTC: 0 },
     internal.reports.sendReportsToAllUsers
   )
   export default crons
   ```
